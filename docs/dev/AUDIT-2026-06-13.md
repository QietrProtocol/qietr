## §1 — x402 conformance (HIGHEST RISK, core to the pitch)

All findings in `qietr-sdk/src/x402.ts` unless noted. Wiring: `qietr-sdk/src/sdk.ts:447-481` (wrapFetch), `:294-441` (pay), `:470-481` (payMicro).

- **[CRITICAL] Not real x402 on the wire.** Header payload is a flat `{scheme,network,asset,payer,payee,amount,signature,invoiceId}` blob (`x402.ts:125-135`) — **no `x402Version`, no `payload` envelope**. Real x402 = base64 `{x402Version, scheme, network, payload}`. A spec server cannot decode it. → Implement the real envelope (per D1).
- **[CRITICAL] Inverted settle model.** SDK does the full on-chain withdraw **first** (`sdk.ts:416`), then ships the tx signature as a "receipt." No facilitator verify→settle. If retry fails, **user paid and gets nothing** (acknowledged in comment `x402.ts:147-148`). → Add facilitator path (per D1).
- **[CRITICAL] Wrong requirements field.** Reads `requirement.amount`; x402 v1 field is **`maxAmountRequired`**. Against a real v1 server `BigInt(undefined)` **throws outside the try** (`x402.ts:106`) and escapes wrapFetch. → Parse `maxAmountRequired` (+ v2 `amount`), keep everything `bigint`, move parse inside guarded try.
- **[CRITICAL] Amount semantics wrong + float round-trip.** `payMicro` converts atomic units back to float (`sdk.ts:478` `Number(paymentMicro)/10**6`) then re-quantizes (`sdk.ts:298`). And `exact` scheme requires transfer == required, but design pays a fixed denomination + keeps change → not "exact." → keep bigint end-to-end; reconcile with exact semantics.
- **[MAJOR] No max-amount / payTo guard.** wrapFetch auto-pays whatever a 402 asks, to whoever it names, no ceiling (`x402.ts:96-118`). **Worst foot-gun for an automated agent.** → add mandatory `maxAmountMicro` + optional `payTo` allowlist to `WrapFetchOptions` (`x402.ts:40-64`). *(Cheapest high-value fix — ~1 hr.)*
- **[MAJOR] No replay/nonce/resource binding.** Receipt has no nonce/timestamp/validBefore, not bound to URL/body → replayable (`x402.ts:125-134`).
- **[MAJOR] No scheme/network negotiation.** Hardcoded `"exact"` + exact-string network match (default `"solana-mainnet"`, `x402.ts:66,93-98`); real labels are `solana:mainnet`/`solana-devnet`/CAIP-2. `asset` echoed but never validated against configured USDC mint → would pay requirements in some other SPL token.
- **[MAJOR] Silent failure swallows all errors.** Every branch `return lastRes` (raw 402); `catch (e)` discards `e` (`x402.ts:115`). No way to tell "no payment attempted" from "proof failed" from "out of funds." → propagate typed errors.
- **[MAJOR] No timeouts.** No `AbortSignal` on base or retry fetch (`x402.ts:79,145`); ignores `maxTimeoutSeconds`. Combined with pre-settlement → paid-and-stuck on hang.
- **[MINOR] Multi-pay loop hazard.** `while (status===402 && attempts<maxRetries)` with no "already paid this request" tracking → `maxRetries>1` + misbehaving merchant = repeated on-chain withdraws (`x402.ts:83-84`).
- **[MINOR] Dead branch + double `getFeePayer()`** (`x402.ts:121-123`); `payer` in receipt is the **fee-payer** (relayer in gasless), not the real payer (`x402.ts:121-129`).
- **[MINOR] Zero wrapFetch tests.** Only a `PaymentRequiredError` name check exists (`test/edge-cases.test.mjs:119-135`). The most security-sensitive module is untested. → add parse/pay/retry/re-fail/over-amount tests.

## §2 — ZK / privacy / soundness

Circuit `qietr-circuits/circuits/qietr_payment.circom`; program `qietr-pool/programs/qietr_pool/src/{lib,verifier,merkle,state,dev_vk}.rs`; SDK `prover.ts/note.ts/merkle.ts`.

- **[CRITICAL] §2.1 Change commitments break the anonymity set.** Circuit allows partial spend minting a **non-tier change note into the same tree in the same tx** (`qietr_payment.circom:166-179`, `lib.rs:281` `tree.append(change_commitment)`). Observer links "spend nullifier X, pays 7.3, new leaf same tx" → change note + its future lineage trivially linkable. README's "Privacy set: every depositor" is **false in practice** for partial spenders. Tornado has no change for this reason. → **D2 decision.**
- **[CRITICAL] §2.3 Dev VK is the live verifier.** `dev_vk.rs` labeled *"DEV KEY. Do not deploy."*, single-contributor `pot14` setup → trapdoor holder can **forge any proof → drain vaults**. Only VK the program uses (`verifier.rs:21,52`, `lib.rs:192/193`). → external ceremony (§0b) + §2.4.
- **[CRITICAL] §2.4 VK-hash check is decorative.** `config.verifying_key_hash` is written (`lib.rs:58,311`) and there's a whole `queue_vk_upgrade`/`apply_vk_upgrade` timelock, but **`withdraw` never enforces the hash** against the embedded VK — verification always uses the hardcoded const. So the website's "48-hour timelock on circuit upgrades" is currently **false**. → make `withdraw` gate on the stored hash so the timelock isn't theater.
- **[MAJOR] Lossy 3-bit recipient masking.** `pubkey_to_field` does `out[0] &= 0x1f` (`lib.rs:342-346`); on-chain compare `lib.rs:163-166`. **8 pubkeys collide** to one field element → a colluding relayer with a vanity-ground colliding key can redirect payment. → hash full pubkey (Poseidon/keccak) reduced mod p instead of bit-clearing.
- **[MAJOR] Recipient/amount soundness rests on single on-chain equality checks** (no real in-circuit binding: `circom:185-186 recipientBound` is a no-op; amount tier-bind only at `lib.rs:188-190`). Correct today, brittle — guard against denom misconfig / deposit-withdraw tree divergence.
- **[MAJOR] 30-root stale window leaks deposit-time upper bound** (`state.rs:10 ROOT_HISTORY_LEN=30`, root is public signal). Narrows anonymity set on sparse tiers (0.1, 100). → document; consider larger window.
- **[MINOR] SDK self-check reads wrong public-signal index.** Authoritative order (from `build/qietr_payment.sym`): `[amount, root, nullifierHash, recipient, paymentAmount, changeCommitment]`. `sdk.ts:374` compares `publicSignals[2]` (= nullifierHash) against `pubkeyToField(args.to)` — checking the wrong field. Harmless today (on-chain check authoritative) but a **reorder landmine**. Also `verifier.rs:8-13`, `pubkey.ts:6`, `qietr-circuits/README.md:30`, `docs/dev/11-trusted-setup.md:15-16` all disagree on order — clean up.
- **[MINOR] Note crypto OK; tune params.** Argon2id `t=3,m=64MiB` + AES-256-GCM, fresh nonce/salt per encrypt (`note.ts:18-23,117-118`) — sound. For spendable-fund secrets raise to `t≥3, m≥256MiB`; bind version/magic into GCM AAD.

## §3 — Relayer (MONEY PATH — not safe to expose)

`qietr-relayer/src/{tx-validation,policy/auth,policy/rate-limit,kora,config}.ts`, `routes/submit.ts`, `index.ts`.

- **[CRITICAL] No economic validation.** `decodeAndValidateWithdraw` (`tx-validation.ts:43-104`) checks discriminator + fee-payer only — **never verifies its fee is paid, never checks own SOL balance, no spend cap** → drainable by repeated zero-fee withdrawals. Deposit path same gap (`tx-validation.ts:137-221` never decodes transfer amount / fee-ATA dest). → enforce fee-is-collected + balance check + per-window spend cap.
- **[CRITICAL] Promised replay protection doesn't exist.** README §4 claims "30s window, same sig rejected" — no such code in `submit.ts`; `kora.ts:39-42` injects fresh blockhash, defeating expiry. → implement real replay defense.
- **[MAJOR] Auth optional + 0.0.0.0.** `auth.ts:7-10` allows all if `API_KEY` unset; `config.ts:74` default `HOST=0.0.0.0`, no warning; `auth.ts:22` uses `!==` (timing-attackable). → mandatory auth + `timingSafeEqual` + warn on bind.
- **[MAJOR] No outbound timeouts** (`kora.ts:40,49,89-93`, `submit.ts:79`, `sanctions.ts:24`). **Sanctions fail open** on load error (`index.ts:52-58`); screens only one side per route. **Rate-limit keys `req.ip` with trustProxy off** → one bucket behind Cloudflare (`rate-limit.ts:39`); in-memory only, no global spend cap.

## §4 — Indexer

- **[CRITICAL] `/merkle-proof` O(n) DoS.** Rebuilds entire Poseidon tree per request, no LIMIT, no cache headers (`api/src/routes/merkle-proof.ts:67-88`, `poseidon-tree.ts:64-82`). → cache + incremental tree.
- **[MAJOR] Geyser can panic the validator.** `decode.rs:95-97` indexes `root_history[u8]` unbounded vs `ROOT_HISTORY_LEN=30`; `writer.rs:64-69` `.expect()` in `on_load`. → bounds-check + return `GeyserPluginError`.
- **[MAJOR] No pg pool error listener / Fastify error handler** (`api/src/db.ts`) → idle-client error crashes process.
- **[MINOR] Racy leaf-index** (`geyser-plugin/src/lib.rs:300-333` + `notify_transaction` on *processed*, forks double-count; `ON CONFLICT DO NOTHING` silently drops → corrupt tree). **NoTls Postgres** (`writer.rs:13,209-213`). Input validation gap `poseidon-tree.ts:107-118` (accepts negatives / above field prime). Metrics `TODO` `writer.rs:79`.
